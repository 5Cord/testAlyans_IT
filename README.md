### Исходное задание

Приложение для работы с полигонами на карте мира в режиме глобуса: пользователь вводит название и координаты, полигон проверяется на пересечение с уже сохранёнными и либо сохраняется, либо попадает в таблицу отклонённых. Отдельное требование: корректная работа с полигонами, пересекающими антимеридиан (180-й меридиан).

## Описание решения

### Использованные технологии и инструменты

1. [React](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
2. [MapLibre GL JS](https://maplibre.org/) - карта в режиме глобуса
3. [TanStack Query](https://tanstack.com/query/latest) - серверное состояние и кэш
4. [Zustand](https://zustand.docs.pmnd.rs/) - выбранный на карте полигон
5. [Turf.js](https://turfjs.org/) - геометрия в mock-режиме (в проде пересечения считает backend2)
6. [React-Toastify](https://fkhadra.github.io/react-toastify/introduction/) - уведомления
7. [Vite](https://vite.dev/) + [Vitest](https://vitest.dev/) + [ESLint](https://eslint.org/)

Структура кода построена по Feature-Sliced Design: `app` / `widgets` / `features` / `entities` / `shared`.

### Описание Backend-части

Бэкенд лежит в отдельном репозитории `testAlyans_IT_beckend` и поднимается через docker compose:

- `backend1` (Django + DRF + PostGIS, порт 8000) - основное API: список полигонов, создание, список отклонённых;
- `backend2` (порт 8001) - внутренний сервис проверки пересечений, к нему обращается только `backend1`;
- `db` - PostgreSQL с расширением PostGIS, общая для обоих сервисов.

Swagger не поднят; посмотреть API можно через browsable-интерфейс DRF на http://localhost:8000/api/.

Фронт использует три эндпоинта: `GET /api/territories/` (список полигонов, GeoJSON FeatureCollection), `POST /api/territories/` (создание, тело `{name, coords: [[lng, lat], ...]}`), `GET /api/rejected/` (полигоны, не прошедшие проверку пересечений).

### Описание Frontend-части

1. Форма "Новый полигон":

- Поля названия, широты и долготы; кнопка "Добавить" дописывает точку в общий список координат (textarea), который можно править и руками.
- Полная валидация до отправки: название, диапазоны координат, минимум 3 точки.
- По Submit название и координаты уходят на бэкенд; при успехе форма очищается.

2. Карта (MapLibre, глобус):

- Все сохранённые полигоны отрисованы на глобусе; клик по полигону или по строке таблицы открывает его крупно с подписью-названием.
- Полигоны через антимеридиан рисуются корректно, без "простыни" через весь глобус: перед отрисовкой долготы разворачиваются так, чтобы соседние точки отличались меньше чем на 180°.
- Для отклонённого полигона показывается конфликт: сам полигон красным, пересекаемые оранжевым, область/линия/точки пересечения - по данным backend2 (`intersection_coords` из ответа API, фронт их не пересчитывает).

3. Таблица полигонов:

- Колонки: название, координаты, пересечение антимеридиана (True/False).
- Клик по строке открывает полигон на карте.

4. Таблица пересечений:

- Отклонённые полигоны с указанием, с чем они пересекаются.
- Клик по строке показывает конфликт на карте.

5. Поиск по названию:

- Подстрока без учёта регистра, с задержкой 300 мс при вводе.
- Единственное совпадение открывается на карте сразу, несколько - списком.

## Запуск

```bash
npm install
npm run dev
```

Дев-сервер поднимается на http://localhost:5173. Для работы с данными нужен запущенный бэкенд:

```bash
docker compose up -d --build
```

### Через Docker

```bash
docker compose up -d --build
```

Прод-сборка отдаётся nginx на http://localhost:3000. Образ собирается в два этапа: node собирает статику, в итоговый слой попадает только nginx с содержимым `dist/`.

Запросы к API по умолчанию идут относительными путями, и nginx проксирует `/api/` на `host.docker.internal:8000` - то есть на бэкенд, поднятый на хост-машине. Так фронт и API живут на одном origin и CORS не нужен вовсе.

Если бэкенд крутится по другому адресу, есть два пути: поменять `proxy_pass` в `nginx.conf`, либо вшить прямой адрес в бандл на этапе сборки (тогда бэкенд должен разрешать origin фронта в CORS):

```bash
docker build --build-arg VITE_API_URL=http://api.example.com -t polygons-front .
docker run -d -p 3000:80 polygons-front
```

В compose `VITE_API_URL` берётся из переменной окружения (или `.env` рядом), по умолчанию пустая - как раз вариант с прокси.

### Переменные окружения

| Переменная | По умолчанию | Что задаёт |
|---|---|---|
| `VITE_API_URL` | `http://localhost:8000` (в Docker - пусто, работает nginx-прокси) | адрес backend API |

Задаётся через `.env` в корне (образец - `.env.example`) или переменную окружения; в Docker - build-arg. WebSocket-эндпоинта нет, списки обновляются обычными HTTP-запросами после действий пользователя.

### Сборка и тесты

```bash
npm run build     # прод-сборка в dist/
npm run preview   # посмотреть собранное локально
npm test          # юнит-тесты (vitest)
npm run lint      # eslint
```

## Ошибки, которые показывает интерфейс

Под полями формы, до отправки на сервер:

- "Введите название полигона" - название пустое;
- "Полигон с таким названием уже есть" - имя занято (сравнение без учёта регистра);
- "Широта должна быть числом", "Широта должна быть от -90 до 90";
- "Долгота должна быть числом", "Долгота должна быть от -360 до 360" - диапазон шире привычного ±180 намеренно: в примере из задания есть долгота 204.96; бэкенд принимает те же пределы, а к ±180 значения нормализуются уже внутри;
- "Строка N: ожидается пара "широта, долгота"", "Строка N: координаты должны быть числами" и построчные проверки диапазонов - для списка координат в textarea;
- "Нужно минимум 3 точки для полигона".

Всплывающие уведомления по ответу бэкенда:

- "Полигон "..." сохранён" - успех, форма очищается, список обновляется;
- "Полигон "..." отклонён: пересекается с существующими" - бэкенд нашёл пересечение; запись появляется в таблице пересечений, клик по строке показывает конфликт на карте;
- если бэкенд вернул осмысленную причину (поле `detail`, например "Территория с таким именем уже существует"), она показывается в toast как есть;
- "Не удалось сохранить полигон. Попробуйте ещё раз" - сетевой сбой или ошибка бэкенда без расшифровки.

В списках и поиске: "Загрузка..." во время запроса, "Пока нет ни одного полигона" и "Отклонённых полигонов нет" при пустых данных, "Ничего не нашлось" при пустом результате поиска.
